1 # apriltag_ros
2 
3 `apriltag_ros` is a Robot Operating System (ROS) wrapper of the [AprilTag 3 visual fiducial detector](https://april.eecs.umich.edu/software/apriltag.html). For details and tutorials, please see the [ROS wiki](http://wiki.ros.org/apriltag_ros).
4 
5 `apriltag_ros` depends on the latest release of the [AprilTag library](https://github.com/AprilRobotics/apriltag). Clone it into your catkin workspace before building.

11 ## Quickstart
12 
13 Starting with a working ROS installation (Kinetic and Melodic are supported):
14 ```
15 export ROS_DISTRO=melodic # Set this to your distro, e.g. kinetic or melodic
16 source /opt/ros/$ROS_DISTRO/setup.bash # Source your ROS distro
17 mkdir -p ~/catkin_ws/src # Make a new workspace
18 cd ~/catkin_ws/src # Navigate to the source space
19 git clone https://github.com/AprilRobotics/apriltag.git # Clone Apriltag library
20 git clone https://github.com/AprilRobotics/apriltag_ros.git # Clone Apriltag ROS wrapper
21 cd ~/catkin_ws # Navigate to the workspace
22 rosdep install --from-paths src --ignore-src -r -y # Install any missing packages
23 catkin build # Build all packages in the workspace (catkin_make_isolated will work also)
24 ```
25 See the [ROS wiki](http://wiki.ros.org/apriltag_ros) for details and tutorials.

27 ## Contributing
28 
29 Pull requests are welcome! Especially for the following areas:
30 
31 - Publishing of the AprilTag 3 algorithm intermediate images over a ROS image topic (which AprilTag 3 already generates when `tag_debug==1`)
32 - Conversion of the bundle calibration script from MATLAB to Python
33 - Extend calibration to support calibrating tags that cannot appear simultaneously with the master tag, but do appear simultaneously with other tags which themselves or via a similar relationship appear with the master tag (e.g. a bundle with the geometry of a cube - if the master is on one face, tags on the opposite face cannot currently be calibrated). This is basically "transform chaining" and potentially allows calibration of bundles with arbitrary geometry as long as a transform chain exists from any tag to the master tag
34 - Supporting multiple tag family detection (currently all tags have to be of the same family). This means calling the detector once for each family. Because the core AprilTag 2 algorithm is the performance bottleneck, detection of `n` tag families will possibly decrease performance by `1/n` (tbd if this still holds for v3)
